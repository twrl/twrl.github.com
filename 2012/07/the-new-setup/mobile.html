About This New Setup

So, this new setup is more or less as I want it. I somewhat prematurely put it live on Github pages on Saturday, even though I’m continuing to tinker with it.

I started off using Jekyll-Bootstrap. This was a massive timesaver. I’ve used Liquid before, so the Jekyll learning curve was very easy for me, but if you haven’t then I recommend Jekyll-Bootstrap as a way of easing the transition. The templates and includes that Jade has in there are fairly adaptable, and you can learn quite a bit just from examining them.

Following the advice in Charlie Park’s blog I’ve got Jekyll running locally, and I’m pushing the compiled static site onto Github. This gives me a free hand to have plugins, non-Github-supported configuration options, and otherwise be tampering with Jekyll.

I’m using Kramdown and Coderay for rendering. Getting Pygments to work reliably on my Mac was just too much of a headache, plus I like Kramdown’s language extensions anyway. At least on my Mac, I had to set Kramdown’s entity_output to symbolic, otherwise it was garbling apostrophes and other entities. No other problems there though.

I’m using Jekyll’s latent semantic indexing to generate the related posts list, but I’m not convinced about it. I really don’t want to maintain the list manually if I can avoid it, so next on the agenda is to investigate other options for that. I also plan to refactor the templates to improve the quality of the semantics in them, and because I’ve ended up with a small mess of css files.

My only grumble is Liquid’s inability to sort an array before iterating over it. So far I’ve managed to work around it, but it is a nuisance.
